# Chapter 10: Adoption Stages

## Core Idea
CSDM adoption is incremental. Foundation, Crawl, Walk, Run, and Fly stages sequence governance and modeling capabilities so value grows without attempting a big-bang migration.

## Frameworks Introduced
- **Foundation → Crawl → Walk → Run → Fly**: Maturity path for CSDM adoption.
  - When to use: Planning implementation or assessing current maturity.
  - How: Complete prerequisites, choose outcomes, add model scope, validate value, then advance.

## Key Concepts
- **Foundation**: Establish governance, ownership, naming, lifecycle, and common data.
- **Crawl**: Build essential application/service visibility and basic relationships.
- **Walk**: Expand service modeling, ownership, and product use.
- **Run**: Improve operational dependency quality and cross-product outcomes.
- **Fly**: Optimize integrated lifecycle, automation, and enterprise use cases.

## Mental Models
- Adopt for measurable use cases, not maturity labels.
- Progress only when current-stage data is governed and producing value.
- Treat adoption as continuous product work.

## Anti-patterns
- **Big-bang migration**: High risk, unclear value, weak ownership.
- **Skipping Foundation**: Later automation scales bad data.
- **Completing tables instead of outcomes**: Population percentage alone is not success.

## Worked Example
An organization first assigns owners and normalizes application records. It then models critical application services for incident impact, expands to business services and offerings, improves automated discovery, and finally integrates strategy-to-runtime traceability.

## Key Takeaways
1. Start with governance and use cases.
2. Sequence adoption by dependencies.
3. Measure outcomes at every stage.
4. Automate only after model quality is stable.
